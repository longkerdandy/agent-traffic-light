# AI 状态灯客户使用说明

这是一套给 Codex / Claude 使用的桌面状态灯产品。  
客户拿到产品后，只需要接好 ESP32 灯板，然后按下面步骤操作即可。

## 一、你会看到哪些文件

- `00-双击这里-一键安装环境.cmd`
  第一次使用时运行。自动安装 Python 依赖，并配置 Codex hook。
- `05-双击这里-启动完整系统.cmd`
  一键启动网页面板和串口桥。
- `06-双击这里-手动测试灯效.cmd`
  手动测试红黄绿灯是否正常。
- `04-双击这里-一键自检.cmd`
  检查电脑环境是否正常。

## 二、第一次使用怎么做

### 第 1 步：连接硬件

把 ESP32-C3 状态灯插到电脑 USB 口。

当前交付硬件的灯线定义：

- 红灯：`GPIO0`
- 黄灯：`GPIO2`
- 绿灯：`GPIO1`

系统会自动检测客户电脑上的 ESP32 串口号，不要求串口号与开发电脑一致。

### 第 2 步：双击安装环境

双击：

`00-双击这里-一键安装环境.cmd`

它会自动做这些事情：

- 检查并安装 Node.js
- 检查并安装 Python
- 安装 Python 串口依赖 `pyserial`
- 自动为当前项目安装 Codex hook

如果电脑是第一次安装 Node.js 或 Python，安装结束后建议再运行一次 `00-双击这里-一键安装环境.cmd`，确认全部完成。

### 第 3 步：做一次手动测试

双击：

`06-双击这里-手动测试灯效.cmd`

看到菜单后，输入数字测试：

- `1`：空闲
- `2`：思考
- `3`：忙碌
- `4`：成功
- `5`：等待确认
- `9`：错误

如果灯能跟着变化，说明硬件和串口都正常。

## 三、日常使用怎么做

### 方式 1：一键启动完整系统

双击：

`05-双击这里-启动完整系统.cmd`

它会自动：

- 启动网页面板
- 启动串口桥
- 自动检测 ESP32 串口号
- 自动打开浏览器页面

网页地址是：

`http://127.0.0.1:8787`

### 方式 2：直接使用 Codex

如果你已经运行过一次安装脚本，那么之后只要 Codex 产生 hook 事件：

- 网站会自动被拉起
- 串口桥也会自动被拉起
- ESP32 会自动检测串口并跟随状态变化

## 四、灯的状态说明

- `idle`：绿灯呼吸
- `thinking`：红黄绿跑马灯
- `busy`：黄灯慢闪
- `success`：绿灯常亮 5 秒后回空闲
- `wait_confirm`：黄灯常亮
- `confirm`：黄灯双闪提醒
- `waiting`：黄灯呼吸
- `wait`：黄绿交替慢闪
- `error`：红灯快闪

## 五、如果不能用，先做这 3 件事

### 1. 运行一键自检

双击：

`04-双击这里-一键自检.cmd`

如果有 `[FAIL]`，先解决失败项。

### 2. 确认灯板能手动测试

双击：

`06-双击这里-手动测试灯效.cmd`

如果手动测试都没反应，先检查：

- USB 线是否正常
- ESP32 是否插好
- 电脑是否识别到开发板

### 3. 关闭占用串口的软件

如果网页正常但灯不动，常见原因是串口被别的软件占用。  
请先关闭这些软件后再试：

- Arduino 串口监视器
- 其他串口工具
- 之前打开的测试脚本窗口

如果更换 USB 接口后串口号发生变化，重新运行：

`05-双击这里-启动完整系统.cmd`

串口桥会重新扫描并连接当前 ESP32 设备。

## 六、客户最常用的操作建议

推荐客户记住这两个文件就够了：

- 第一次使用：`00-双击这里-一键安装环境.cmd`
- 平时使用：`05-双击这里-启动完整系统.cmd`

如果只是检查灯是否正常：

- `06-双击这里-手动测试灯效.cmd`
