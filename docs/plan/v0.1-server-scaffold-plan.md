# v0.1 Server Scaffold Plan

## Goal

Deliver the **server-side project framework** for `agent-traffic-light` with no business logic yet. This milestone focuses on repository structure, build tooling, code quality enforcement, CI/CD, IDE configuration, and GitHub security scanning so that future feature work starts from a clean, automated baseline.

## Scope

### In scope for v0.1

- .NET solution and server project skeleton (empty ASP.NET Core host).
- Shared contracts project placeholder.
- Test project skeleton with a single placeholder test.
- Code quality tooling: EditorConfig, .NET analyzers, `dotnet format`.
- GitHub Actions workflows: build/test/format CI, CodeQL security analysis.
- Dependabot configuration for NuGet and GitHub Actions.
- VSCode workspace configuration (extensions, settings, tasks, launch).
- `.gitignore` update to allow committed VSCode settings.

### Out of scope for v0.1

- Any business logic: hook handling, session store, state machine, serial port control, dashboard.
- Kimi Code agent client code.
- Packaging, Windows service, systemd installation.
- Documentation beyond this plan and inline config comments.

## Deliverables

### 1. .NET Solution & Server Skeleton

| File | Purpose |
|------|---------|
| `server/global.json` | Pin the .NET SDK version (`10.0.x`). |
| `server/Directory.Build.props` | Shared MSBuild properties: nullable, implicit usings, analyzers, warnings-as-errors. |
| `server/agent-traffic-light.sln` | Solution referencing server, contracts, and test projects. |
| `server/AgentTrafficLightServer/AgentTrafficLightServer.csproj` | ASP.NET Core empty project targeting `net10.0`. |
| `server/AgentTrafficLightServer/Program.cs` | Minimal placeholder host (no endpoints besides default). |

| `server/AgentTrafficLightServer/appsettings.Development.json` | Development logging configuration. |
| `server/AgentTrafficLightServer/Properties/launchSettings.json` | Launch profiles for Kestrel on `http://127.0.0.1:8787`. |
| `server/AgentTrafficLight.Contracts/AgentTrafficLight.Contracts.csproj` | Class library placeholder for shared DTOs. |
| `server/AgentTrafficLightServer.Tests/AgentTrafficLightServer.Tests.csproj` | xUnit test project referencing the server. |
| `server/AgentTrafficLightServer.Tests/PlaceholderTests.cs` | One passing placeholder test so CI has something to run. |

### 2. Code Quality Tooling

- **EditorConfig** (`.editorconfig`) defines C#, JSON, TOML, Markdown, and shell script formatting rules.
- **.NET Analyzers** enabled via MSBuild with `AnalysisLevel` set to `latest-recommended`.
- **TreatWarningsAsErrors** on Release builds; warnings allowed locally in Debug for faster iteration.
- **Style consistency**: `dotnet format` invoked in CI to verify no formatting regressions.

### 3. GitHub Actions

| Workflow | File | Purpose |
|----------|------|---------|
| CI | `.github/workflows/ci.yml` | Build, test, and verify formatting on every push and PR. |
| CodeQL | `.github/workflows/codeql.yml` | Run GitHub CodeQL C# security analysis weekly and on PRs. |

### 4. GitHub Security Checks

- **Dependabot** (`.github/dependabot.yml`) monitors `nuget` and `github-actions` ecosystems weekly.
- **CodeQL** workflow scans for vulnerabilities in C# code.
- Branch protection rules are recommended (manual repo setting) but not part of committed files.

### 5. VSCode Configuration

| File | Purpose |
|------|---------|
| `.vscode/extensions.json` | Recommend C# Dev Kit, EditorConfig, and GitHub Actions extensions. |
| `.vscode/settings.json` | Format on save, default formatter, file exclusions, analyzer output. |
| `.vscode/tasks.json` | `build`, `test`, and `watch` tasks. |
| `.vscode/launch.json` | Debug launch profile for the ASP.NET Core server. |

## Risks

- .NET SDK is not installed in the current environment, so local validation must wait until it is available; file creation can still proceed because all deliverables are text-based.
- Warnings-as-errors may feel strict early on, but it is scoped to Release/CI builds.
- CodeQL requires GitHub Advanced Security to be enabled for private repositories.
